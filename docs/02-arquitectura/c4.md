# Modelo C4

El modelo **C4** describe una arquitectura en niveles de zoom: Contexto → Contenedores →
Componentes. Aquí mostramos los dos primeros, suficientes para este proyecto.

## Nivel 1 — Contexto

¿Quién usa el sistema y con qué se relaciona?

```mermaid
flowchart TB
    EST["👩‍🎓 Estudiante / Verificador<br/>(persona)"]
    EMI["🏫 Emisor / Institución<br/>(persona)"]
    SYS["📜 Sistema Registro de Certificados<br/>(DApp + contrato)"]
    ETH["⛓️ Ethereum<br/>(sistema externo)"]
    MM["🦊 MetaMask<br/>(sistema externo)"]

    EMI -->|emite / revoca certificados| SYS
    EST -->|verifica certificados| SYS
    SYS -->|lee / escribe estado| ETH
    SYS -->|firma transacciones| MM
    MM --> ETH
```

## Nivel 2 — Contenedores

¿Qué piezas desplegables componen el sistema?

```mermaid
flowchart TB
    subgraph SYS["Sistema Registro de Certificados"]
        FE["DApp Frontend<br/>HTML + ethers.js v6<br/>[contenedor: web estática]"]
        SC["Contrato RegistroCertificados<br/>Solidity 0.8.24<br/>[contenedor: on-chain]"]
        DJ["deployment.json<br/>dirección + ABI<br/>[archivo de enlace]"]
    end
    MM["🦊 MetaMask"]
    RPC["🔌 Nodo RPC<br/>Alchemy / Infura / Hardhat"]
    BC[("⛓️ Blockchain<br/>estado + eventos")]

    FE -->|lee dirección y ABI| DJ
    FE -->|llama funciones| MM --> SC
    FE -->|lecturas view| RPC --> SC
    SC --> BC
```

## Mapeo a carpetas del repo

| Contenedor | Dónde vive |
|------------|-----------|
| DApp Frontend | `frontend/` |
| Contrato | `contracts/RegistroCertificados.sol` |
| deployment.json | `frontend/deployment.json` (lo genera `scripts/deploy.js`) |
| Pruebas | `test/RegistroCertificados.test.js` |
| Infra nube | `infra/terraform/` |
